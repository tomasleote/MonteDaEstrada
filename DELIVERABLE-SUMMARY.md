## Prompt Engineering Approach

### Techniques Applied

1. **Chain-of-Thought Structure**
   - Context before detail (executive brief before specifications)
   - Decision frameworks before execution
   - Strategic principles before tactical steps

2. **Few-Shot Learning**
   - Concrete examples for every major concept
   - Template structures (not just descriptions)
   - YAML snippets with comments
   - Decision trees with multiple options

3. **Detailed Prompting**
   - Objective + Scope + Deliverable + Validation per section
   - "What to accomplish" + "How to accomplish" + "How to validate"
   - Specific vs. vague language throughout

4. **Role-Based Prompting**
   - Different perspectives for different team members
   - Backend, strategist, PM, content, frontend, QA each have clear path
   - Quick reference for rapid onboarding

5. **Constraint-Based Optimization**
   - 97 images, 6 pages, 5-7 days - all constraints explicit
   - Success criteria tied to constraints
   - Timeline realistic and phase-based

### Context Architecture

```
Strategic Context
├── Touril aesthetic (colors, typography, spacing)
├── Project constraints (React, SCSS, mobile-first)
├── Image inventory (97 total, 7 categories)
└── Team capabilities (experienced developers assumed)

Tactical Context
├── CMS specifications (Decap CMS structure)
├── Page specifications (6 pages, each with image specs)
├── Component specifications (hero, card, gallery, etc.)
└── Data specifications (CMS fields, alt text, captions)

Execution Context
├── Timeline (5-7 days, 8 phases)
├── Roles (backend, strategist, PM, content, frontend, QA)
├── Procedures (step-by-step checklists)
└── Validation (completeness, functionality, accessibility, performance)
```
