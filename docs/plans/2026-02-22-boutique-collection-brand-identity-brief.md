# Boutique Collection Brand Identity
## A Comprehensive Design Brief for Premium Portuguese Hospitality

**Date**: February 22, 2026
**Status**: Final Brand Identity Brief
**Target Audience**: Design-conscious travelers (25-50) seeking quiet luxury and authentic experiences
**Scope**: Visual identity system for a curated collection of two boutique guesthouses in the Alentejo region, Portugal

---

## Executive Summary

The Boutique Collection represents a **complete departure from traditional Portuguese rural tourism aesthetics**. This is a **modern, editorial, architecture-forward brand** positioned for the globally-minded traveler who values design, authenticity, and intentionality.

**Key Differentiators:**
- **Sophisticated, not rustic** – Contemporary luxury rooted in place
- **Curated, not generic** – Editorial sensibility guides all decisions
- **Architectural clarity** – Geometric precision, generous whitespace, photography as hero
- **Unified yet distinct** – One DNA across properties, subtle personality variations per location
- **Design-forward digital** – Website feels like a high-performance architecture magazine

This brand identity establishes a **new standard for Portuguese boutique hospitality**, positioning the collection as a destination for travelers who seek authentic spaces designed with intention.

## 3. Typography System

### Font Stack

#### Display/Headlines: Basis Grotesque
- **Category**: Geometric sans-serif with personality
- **Character**: Contemporary, architectural, distinctive without being trendy
- **Weights Used**: Regular (400), Medium (500)
- **Usage**: Page titles (H1), section headers (H2), prominent UI elements
- **Why**: Gives the brand a contemporary, design-forward identity; signals "new generation of luxury hospitality"

#### Body/UI: Inter
- **Category**: Modern sans-serif, highly optimized for screens
- **Character**: Refined, minimal, exceptionally legible
- **Weights Used**: Regular (400), Medium (500), SemiBold (600)
- **Usage**: Body text, UI labels, navigation, form inputs, labels
- **Why**: Pairs perfectly with Basis Grotesque; ensures excellent readability; modern and functional

#### Editorial Accents: GT Sectra
- **Category**: Contemporary serif (NOT traditional or ornate)
- **Character**: Sophisticated, editorial, modern serif that feels design-forward
- **Weights Used**: Regular (400), Medium (500)
- **Usage**: Pull quotes, testimonials, editorial moments, feature callouts (optional, used sparingly)
- **Why**: Adds editorial sophistication without feeling rustic or dated; contemporary serif positioning

### Typography Scale

**Recommended Size Hierarchy:**

| Element | Size | Weight | Letter Spacing | Line Height | Usage |
|---------|------|--------|-----------------|-------------|-------|
| **H1 (Page Title)** | 3.5rem / 56px | 400 | 1px | 1.2 | Main page titles, hero headlines |
| **H2 (Section Header)** | 2.75rem / 44px | 400 | 1px | 1.3 | Section titles, major headers |
| **H3 (Subsection)** | 2rem / 32px | 400 | 0.75px | 1.4 | Subsection headers, emphasis |
| **H4 (Minor Header)** | 1.5rem / 24px | 500 | 0.75px | 1.5 | Card titles, small headers |
| **Body Large** | 1.125rem / 18px | 400 | 0.5px | 1.6 | Feature text, prominent descriptions |
| **Body Regular** | 1rem / 16px | 400 | 0.5px | 1.6 | Main body copy, standard text |
| **Body Small** | 0.875rem / 14px | 400 | 0.5px | 1.6 | Secondary text, captions, fine print |
| **Label/UI** | 0.875rem / 14px | 500 | 0.75px | 1.4 | Form labels, button text, nav items |
| **Caption/Small** | 0.75rem / 12px | 400 | 0.5px | 1.5 | Metadata, timestamps, tiny text |

### Letter Spacing Philosophy

**Why generous letter spacing?**
- Creates **editorial quality** and sophistication
- Improves **readability** on screens
- Provides **breathing room** and visual elegance
- Signals **premium, intentional design** (not generic)
- Makes **uppercase text more legible** and refined

**Standard letter-spacing values:**
- Headlines (H1-H2): `1px` (1 letter-space per 56px title feels perfect)
- Section headers (H3-H4): `0.75px`
- Body text: `0.5px` (subtle, improves readability without being obvious)
- Labels/UI: `0.75px` (matches headers for consistency)
- Caption text: `0.5px`

## 4. Visual Style & Design Principles

### Core Design Philosophy

The visual identity is built on **5 core principles**:

#### 1. Photography as Hero
- High-quality, intentional imagery drives the visual experience
- Images should feature warm, natural light
- Focus on architectural detail, materiality, and authentic spaces
- Text and UI support the imagery; they never compete
- Minimum 40% whitespace on pages to let images breathe

#### 2. Architectural Precision
- Sharp, geometric layouts (0px border-radius everywhere)
- Clean lines, clear structure, evident grid system
- Precision in spacing and alignment
- No decorative flourishes; all visual elements serve function
- Layout should feel like a well-composed architectural drawing

#### 3. Generous Whitespace
- Whitespace is a design element, not wasted space
- Creates sophistication, clarity, and breathing room
- Minimum spacing between elements: 24px (3x base unit)
- Large vertical spacing between sections: 64px+ (8x base unit)
- Horizontal margins: 24-32px on desktop, scales responsively

#### 4. Restrained Color Use
- Neutral palette (creams, browns, grays) dominates
- Clay accent (#B8925F) used **sparingly and strategically**
  - CTAs and primary buttons
  - Key links and interactive elements
  - Accent lines or brackets for emphasis
  - One accent per page maximum (typically header/footer or key CTA)
- No background color changes; stick to cream and off-white
- Let photography provide visual variety

#### 5. Editorial Quality
- Layouts inspired by luxury architecture publications (Wallpaper*, Dwell, Dezeen)
- Copy is carefully crafted, never marketing-speak
- Visual hierarchy is bold and confident
- Typography-driven design (text is design, not just content)
- Every page feels like reading a beautifully-designed magazine article

### Visual Elements Specification

| Element | Specification | Rationale |
|---------|---------------|-----------|
| **Border Radius** | 0px (everywhere) | Sharp, geometric, contemporary feel |
| **Borders** | 1px solid #DDD9D5 (when needed) | Subtle structure, minimal visual weight |
| **Shadows** | None (or extremely subtle) | Flat, modern aesthetic; elevation through whitespace |
| **Opacity Effects** | Used sparingly (hover states, overlays) | Refined interactions without visual chaos |
| **Depth/Layering** | Achieved through spacing and scale, not shadows | Architectural approach to hierarchy |

### Responsive Design Approach

**Mobile-first philosophy:**
- Design base experience for 375px viewport
- Enhance for 768px (tablet) and 1024px+ (desktop)
- All content remains readable at smallest size
- Images scale fluidly; maintain aspect ratios
- Typography scales smoothly (use `clamp()` for fluid sizing)

**Key Breakpoints:**
- Mobile: 375px - 767px
- Tablet: 768px - 1023px
- Desktop: 1024px+
- Wide Desktop: 1440px+

## 6. Motion & Interaction Design

### Animation Philosophy

**Principle:** Motion should be **purposeful, subtle, and refined**. It should enhance usability and delight without calling attention to itself.

**Key Rules:**
- No motion for motion's sake (no decorative animations)
- All transitions should have a clear purpose
- Timing should feel natural, not rushed
- Easing functions should be smooth and organic

### Interaction Timing Standards

| Interaction Type | Duration | Easing | Purpose |
|-----------------|----------|--------|---------|
| **Quick feedback** (hover, focus) | 150ms | ease-out | Immediate response to user action |
| **Standard transition** (page fade, button change) | 300ms | ease-in-out | Balanced, natural feeling |
| **Smooth scroll** (navigation to section) | 400ms | ease-in-out | Guided, intentional movement |
| **Menu open/close** (navigation) | 400ms | ease-out | Smooth entrance/exit |
| **Loading state** (image load, form submit) | 500ms | ease-in-out | Calm, patient feel |

### Specific Interaction Patterns

**Link Hover State:**
```css
/* Default state */
text-decoration: none;
color: #3D3B38 (Deep Brown);

/* Hover state */
color: #B8925F (Clay);
text-decoration: underline;
transition: all 300ms ease-in-out;
```

**Button Hover State:**
```css
/* Default state */
background-color: #B8925F (Clay);
color: #3D3B38 (Deep Brown);

/* Hover state */
opacity: 0.85;
transform: scale(1.02);
transition: all 300ms ease-in-out;
```

**Image Hover State:**
```css
/* On gallery images or image links */
opacity: 1;
filter: brightness(1.05);
transition: opacity 300ms ease-in-out, filter 300ms ease-in-out;
```

**Form Input Focus State:**
```css
/* Default state */
border: 1px solid #DDD9D5 (Warm Gray);

/* Focus state */
border: 1px solid #B8925F (Clay);
outline: none;
box-shadow: none; /* Maintain minimal aesthetic */
transition: border-color 300ms ease-in-out;
```

**Page/Section Transition:**
```css
/* New content fades in subtly */
opacity: 0 → 1;
transition: opacity 300ms ease-in-out;
/* Optional: slight vertical shift for depth */
transform: translateY(10px) → translateY(0);
```

### No-Motion Considerations

Respect user preferences with `prefers-reduced-motion`:
```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

## 11. Implementation Guidelines for Designers & Developers

### Design Handoff

**For UI/UX Designers:**
1. All designs should use Basis Grotesque (display), Inter (body), and GT Sectra (editorial moments only)
2. All color values must use the defined palette (no custom colors)
3. All spacing must use 8px base unit multiples
4. All border-radius: 0px
5. No shadows unless explicitly approved
6. Create components library with shared patterns
7. Document all states (default, hover, focus, active, disabled)

**For Frontend Developers:**
1. Implement as SCSS modules (not Tailwind)
2. Import color palette from `_variables.scss`
3. Use CSS variables for colors for easy property-specific variations
4. Set letter-spacing on base elements (body, h1, h2, etc.)
5. Test all interactions at 300ms timing standard
6. Ensure responsive design works from 375px upward
7. Test color contrast (WCAG AA minimum, AAA preferred)
8. Implement prefers-reduced-motion media query

### Accessibility Requirements

**Minimum Standards (WCAG 2.1 AA):**
- Text contrast ratios: 4.5:1 (normal text), 3:1 (large text)
- All interactive elements: keyboard accessible, visible focus state
- Images: descriptive alt text
- Forms: associated labels, clear error messages
- Headings: semantic hierarchy (h1, h2, h3)
- Motion: respect prefers-reduced-motion preference

**Recommended (WCAG 2.1 AAA):**
- Text contrast: 7:1 (normal), 4.5:1 (large)
- Enhanced color distinction for interactive elements
- Extra visual feedback for interactive states
