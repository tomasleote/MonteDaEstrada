## Naming Convention Guide

All images follow this pattern:
```
[category]-[description]-[number].jpg
```

### Categories:
- `home` - Homepage images
- `quartos` - Room/accommodation images
- `galeria` - Gallery page images
- `atividades` - Activity/festival images
- `redondezas` - Regional attraction images
- `exterior` - Amenity/facility images
- `logos` - Brand logos

### Descriptions:
- `hero` - Hero/header images
- `room` - Individual room photos
- `common-area` - Shared space images
- `amenity` - Amenity/facility images
- `beach` - Beach photos
- `church` - Religious landmarks
- And other descriptive terms

### Numbers:
- Two digits: `01`, `02`, `03`
- For series of related images
- Not used if only one image of type

### Examples:
- `home-hero-monte-exterior.jpg` - Home page hero
- `quartos-room-01.jpg` - First room image
- `redondezas-zambujeira-beach.jpg` - Named location (no number)
- `atividades-festival-sudoeste-02.jpg` - Second festival image

## Adding New Images

### Workflow:
1. **Prepare image**: Optimize to specs (max 2-3 MB, appropriate dimensions)
2. **Add to folder**: `src/assets/images/[category]/[filename].jpg`
3. **Update barrel file**: Add import and export to `src/assets/images/[category]/index.js`
4. **Update JSON**: Add to `src/data/[category].json` if CMS-visible
5. **Update this manifest**: Add to appropriate table
6. **Update components**: Import from barrel file in React

### Checklist:
- [ ] Image file placed in correct category folder
- [ ] Barrel file updated with import and export
- [ ] Alt text is 10-150 characters
- [ ] JSON file updated (if needed)
- [ ] Component imports barrel correctly
- [ ] Manifest.md updated with new image info
- [ ] File follows naming convention
- [ ] Image optimized to spec
